# ⚙️[기능추가][UI] 모바일 접근 시 PC 권장 모달 표시 및 타이틀 반응형 폰트 적용

## 개요

PickerPicker는 키보드 조작 기반 리듬 게임으로 모바일 환경을 공식 지원하지 않는다. 모바일 브라우저로 접근 시 안내 없이 게임 화면이 노출되는 문제를 해결하기 위해, viewport 너비 기준으로 모바일 감지 후 PC 플레이 권장 모달을 표시하는 기능을 추가했다. 게임 진입을 차단하지 않고 확인 후 계속 플레이할 수 있으며, 세션 내 1회만 표시된다. 추가로 모바일 화면에서 넘치던 h1 타이틀 폰트를 반응형으로 수정했다.

## 변경 사항

### 신규 컴포넌트
- `src/components/common/MobileWarningModal.tsx`: PC 플레이 권장 모달 컴포넌트 신규 생성. SVG 모니터 아이콘, 주요 키워드 강조(파란색), "확인" 버튼으로 dismiss

### 기존 컴포넌트 수정
- `src/components/StartScreen.tsx`: h1 타이틀 `text-5xl` → `text-3xl sm:text-5xl` 반응형 폰트 적용
- `src/App.tsx`: 모바일 감지 state(`showMobileWarning`) 및 `handleMobileWarningClose` 핸들러 추가, `MobileWarningModal` 렌더 연결

## 주요 구현 내용

**모바일 감지 로직** (`App.tsx`):
- 초기 렌더 시 `window.innerWidth < 768` 조건으로 모바일 여부 판단
- `sessionStorage` 키(`pickerpicker_mobile_warned`)로 세션 내 중복 표시 방지
- 모달 dismiss 시 sessionStorage에 플래그 저장 → 탭 닫으면 초기화(재방문 시 재표시)

**MobileWarningModal 디자인**:
- 게임 기존 UI 톤(다크 블루, `#00b4ff` 강조)과 통일
- `fixed inset-0 z-50`으로 전체 오버레이, `backdropFilter: blur` 적용
- 게임 차단 없이 "확인" 버튼으로 즉시 dismiss 가능

## 주의사항

- 모바일 감지는 초기 렌더 시점의 `window.innerWidth`만 참조하므로, 리사이즈 이벤트는 별도 처리하지 않음 (의도된 동작)
- 태블릿(768px 이상)은 모달 미표시
