# [버그][Login] PIN 재입력 화면에 이전 PIN이 남아 확인이 불편함

## 개요

신규 회원가입 플로우에서 PIN 설정(`pin-create`) → PIN 확인(`pin-confirm`) 화면으로 전환될 때, 직전에 입력한 PIN 값이 입력창에 그대로 남아 있어 “재입력 확인”의 본래 목적이 흐려지던 문제를 해결한다. `StartScreen.tsx` 내부 `PinView` 컴포넌트가 화면 전환 시 동일 JSX 위치에서 재사용되며 React 재조정(reconciliation)에 의해 `useState('')` 내부 상태가 유지되던 것이 원인이었다. `<PinView>`에 `key={screen}` prop을 부여하여 단계 전환마다 컴포넌트를 강제로 언마운트/재마운트시켜 PIN 입력 state를 초기화한다.

## 변경 사항

### 프론트엔드 (로그인 플로우)
- `src/components/StartScreen.tsx`: PIN 단계 분기 렌더링 부분의 `<PinView>`에 `key={screen}` 추가. `screen` 값(`pin-login` / `pin-create` / `pin-confirm`)이 바뀔 때마다 React가 새 컴포넌트 인스턴스로 인식하여 내부 `pin` state가 초기값(`''`)으로 리셋된다.

## 주요 구현 내용

- 단일 `<PinView>`를 세 가지 모드(`login` / `create` / `confirm`)로 재사용하는 구조는 유지하되, `key`를 통해 단계별로 컴포넌트 정체성을 분리.
- 추가 state 도입이나 부모로의 상태 lifting 없이 1줄 변경으로 해결 — UX 의도(“직접 다시 입력”)와 React 동작이 일치하도록 함.
- PIN 불일치 시 `pin-create`로 복귀하는 기존 흐름(에러 메시지 + 처음부터 재설정)은 그대로 유지된다.

## 주의사항

- `key` 부여로 화면 전환 시 `autoFocus`가 매 단계 새로 적용되므로, 사용자가 PIN 확인 단계 진입 시 자동으로 입력창에 포커스가 잡힌다.
- `pin-confirm` → 이전 단계 복귀(BACK) 후 다시 진입할 때도 입력값이 비어 있는 상태로 시작된다(의도된 동작).
- 추후 PIN 자릿수 변경이나 다단계 인증이 추가될 경우, 동일 패턴(단계 식별자를 `key`로 사용)을 적용하면 동일한 입력 초기화 효과를 얻을 수 있다.
