## 주요 구현 내용

**화면 전환 흐름**

```
home → nickname → pin-create → pin-confirm → 게임 (신규)
home → nickname → pin-login → 게임 (기존)
PIN/닉네임 단계 ← BACK → 이전 단계 역행
```

**PIN 가이드라인 문구** — `pin-create`, `pin-confirm` 두 단계 모두 노란 경고 박스로 표시:
- create: "⚠️ 이 PIN은 다음 로그인 시 필요합니다. 꼭 기억해 두세요!"
- confirm: "⚠️ PIN을 다시 한번 입력하세요. 잊어버리면 재설정이 불가합니다."

**뒤로가기 단계별 복귀** — PIN 단계 BACK → 닉네임, 닉네임 BACK → 홈으로 역행하여 설정/크레딧과 동일한 UX 패턴 적용.

## 주의사항

- `isNewPlayer` prop은 `GameScreen`에 `false` 고정 전달로 변경됨 — 신규 여부 로직이 StartScreen 내부로 흡수되었으므로, 향후 GameScreen에서 신규 여부를 활용하는 로직이 필요하면 별도 처리 필요
- `NicknameModal.tsx`, `PinModal.tsx`는 완전 삭제됨 — 롤백 시 해당 파일 복원 필요
