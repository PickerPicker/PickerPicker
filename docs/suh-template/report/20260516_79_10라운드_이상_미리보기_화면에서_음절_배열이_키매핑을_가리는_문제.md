# [버그][게임] 10라운드 이상 미리보기 화면에서 음절 배열이 키매핑을 가리는 문제

## 개요

STAGE 10 이상에서 본 게임 시작 직전 2초간 표시되는 **리듬 큐 미리보기 화면**에서, 음절 배열이 화면 하단의 키매핑(A/S/D/F · J/K/L/;) 영역을 가려 일부 또는 전부가 보이지 않던 문제를 해결한다.

근본 원인은 `PreviewStage`에서 음절 칸 크기(`w-20 h-20`, 80×80px)와 컨테이너 가로 폭(`max-w-2xl`, 672px)이 스테이지와 무관하게 고정되어 있었기 때문이다. STAGE 10~12는 음절 40개, STAGE 13~15는 48개로 한 줄에 7칸씩 wrap되며 6~7줄로 부풀어, 부모 flex 컨테이너의 `flex-1` 영역을 초과하면서 `shrink-0`인 키매핑 영역을 시각적으로 침범했다.

음절 수에 따라 칸 크기 / 컨테이너 폭 / gap을 모두 동적으로 조정하는 tier 함수를 도입하여, 세로 줄 수를 1~4줄로 제한하고 키매핑이 항상 보이도록 한다.

## 변경 사항

### 프론트엔드 (게임 미리보기 화면)
- `src/components/game/PreviewStage.tsx`
  - `getSyllableLayout(count)` 함수 추가 — 음절 수에 따라 `{ size, maxWidth, gap }` 세 값을 동시에 반환
    - `count ≤ 16`: `w-20 h-20 text-3xl` / `max-w-2xl` / `gap-2` (기존 유지)
    - `count ≤ 32`: `w-16 h-16 text-2xl` / `max-w-4xl` / `gap-2`
    - `count ≤ 48`: `w-12 h-12 text-xl` / `max-w-5xl` / `gap-1.5`
  - 음절 영역 wrapper에 `min-h-0 overflow-hidden` 추가 — 부모 flex가 잘못 분배되더라도 키매핑 영역을 시각적으로 침범하지 않도록 보호

## 주요 구현 내용

- 칸 크기 축소(`w-20` → `w-12`)와 가로 폭 확장(`max-w-2xl` → `max-w-5xl`)을 **함께** 적용하여, 후반 스테이지에서도 한 줄 당 들어가는 칸 수를 늘리면서 동시에 한 칸 면적을 줄여 세로 줄 수가 폭증하는 것을 막는다.
- `gap` 값도 `gap-1.5`로 함께 축소하여 48개(STAGE 13~15) 시나리오에서도 여유 있게 4줄 이내로 표시되도록 한다.
- `KeyboardDisplay`는 변경하지 않음. 기존 `shrink-0` 보장 동작 그대로 유지.
- 스테이지 데이터(`public/rhythm_stages_001_015.json`) 또한 변경하지 않음 — 음절 수 자체는 게임 디자인 의도대로 유지.

## 주의사항

- 향후 음절 수가 48개를 초과하는 스테이지가 추가될 경우, `getSyllableLayout`의 tier를 한 단 더 확장하거나 49개 이상 케이스에 대한 추가 축소 로직(`w-10`, `max-w-6xl` 등)을 도입해야 한다.
- 칸 크기 축소로 후반 스테이지에서는 글자 크기도 `text-xl`까지 작아지지만, 미리보기는 2초간 음절 흐름을 훑어보는 용도이므로 가독성 영향은 제한적이다.
- 본 게임(`PlayStage`)의 노트 트랙 및 글자 크기는 변경 대상이 아니므로 별도 영향 없음.
