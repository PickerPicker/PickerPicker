### ✅ 1. 기본 기능 동작 확인

- [ ] STAGE 1 진입 직전 미리보기에서 음절 16개가 한 칸 80×80(`w-20`) 크기로 표시되는지 확인 (회귀 검증)
- [ ] STAGE 4 (24개) 진입 시 음절 칸이 자동으로 한 단계 작아지는지(`w-16`, 64×64) 확인
- [ ] STAGE 7 (32개) 진입 시 칸 크기 `w-16` 유지 + `max-w-4xl`로 가로가 더 넓어지는지 확인
- [ ] STAGE 10 (40개) 진입 시 칸 크기 `w-12`(48×48), `max-w-5xl`로 한 단계 더 축소 + 확장 적용 확인
- [ ] STAGE 15 (48개) 진입 시에도 음절 배열과 키매핑이 **동시에** 모두 보이는지 확인

### ⚠️ 2. 엣지 케이스

- [ ] STAGE 10 미리보기에서 화면 하단 키매핑(A/S/D/F · J/K/L/;) 8칸이 잘리지 않고 모두 보이는지 확인
- [ ] STAGE 13~15 (48개) 시나리오에서 음절 배열이 **최대 4줄**을 넘지 않는지 확인
- [ ] 음절 영역과 키매핑 영역 사이에 겹침/침범이 없는지 시각적으로 확인
- [ ] 미리보기 2초 표시 후 자동으로 본 게임 화면으로 전환되는 기존 동작이 유지되는지 확인 (`PREVIEW_DURATION_MS = 2000`)
- [ ] 본 게임(`PlayStage`) 진입 후 노트 트랙과 키매핑 영역이 영향 없는지 (변경 대상 아님) 확인

### 🔁 3. 화면 전환 회귀

- [ ] STAGE 1 → 클리어 → STAGE 2 → … → STAGE 10 진행 시, 각 미리보기 화면이 음절 수에 따라 정상적으로 사이즈 변환되는지 확인
- [ ] STAGE 9(32개) → STAGE 10(40개) 전환 시 칸 크기가 `w-16` → `w-12`로 매끄럽게 바뀌는지 확인
- [ ] 게임오버 → "다시 하기" → STAGE 1부터 재시작 시 미리보기 레이아웃이 정상인지 확인
- [ ] 첫 스테이지에서 게임오버 후 다시 하기 시에도 음절 매핑이 재셔플되며 미리보기 레이아웃은 STAGE 1 기준(w-20) 유지되는지 확인 (#75 회귀 검증)

### 🎨 4. UI / UX 확인

- [ ] STAGE 10~15 음절 칸 내부 글자(예: 해/커/톤/장)가 `text-xl`로 작아져도 **육안으로 명확히 판독 가능한지** 확인
- [ ] "리듬 큐" 라벨이 항상 음절 배열 위에 표시되는지 확인
- [ ] 키매핑 영역의 배경(`bg-base-200`)과 음절 영역 사이 시각적 구분이 명확한지 확인
- [ ] 짧은 viewport 높이(브라우저 창을 세로로 줄였을 때)에서도 키매핑이 가려지지 않는지 확인 (`overflow-hidden` + `flex-1` 동작 검증)
