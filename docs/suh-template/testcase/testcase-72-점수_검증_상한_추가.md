### ⚠️ 2. 상한 검증 (이번 변경 핵심)

**score 상한 (`MAX_SCORE = 10,000,000`)**
- [ ] `score=10000000` (경계값 OK) → 200 OK
- [ ] `score=10000001` (경계값+1) → **422 Unprocessable Entity**
- [ ] `score=2147483647` (INT32 MAX, 기존 치팅 사례) → **422**
- [ ] `score=-1` (음수) → **422** (기존 `ge=0` 검증)

**stage 상한 (`MAX_STAGE = 100`)**
- [ ] `stage=100` (경계값 OK) → 200 OK
- [ ] `stage=101` (경계값+1) → **422**
- [ ] `stage=0` → **422** (기존 `ge=1` 검증)
- [ ] `stage=15` (현재 마지막 스테이지) → 200 OK

**combo 상한 (`MAX_COMBO = 9,999`)**
- [ ] `combo=9999` (경계값 OK) → 200 OK
- [ ] `combo=10000` (경계값+1) → **422**
- [ ] `combo=2147483647` (INT32 MAX, 기존 치팅 사례) → **422**
- [ ] `combo=-1` (음수) → **422**
