## 🧪 테스트 케이스: PIN 재입력 화면 입력값 초기화

**이슈**: #73
**담당자**: @Cassiiopeia
**도메인**: Frontend / 로그인 플로우 (`StartScreen.tsx` → `PinView`)

---

### ✅ 1. 기본 기능 동작 확인

- [ ] 신규 닉네임으로 시작 시 닉네임 입력 → `pin-create` 화면 진입
- [ ] `pin-create` 화면에서 4자리 PIN(예: `1234`) 입력 후 [확인] 클릭
- [ ] `pin-confirm` 화면 진입 직후 PIN 입력창이 **비어 있는지** (●●●● 자리표시자만 보이는지) 확인
- [ ] 입력 포커스가 PIN 입력창에 자동으로 잡히는지(`autoFocus`) 확인
- [ ] `pin-confirm` 화면에서 동일 PIN(`1234`) 직접 입력 → [확인] → 계정 생성 + 게임 화면 진입

### ⚠️ 2. 엣지 케이스

- [ ] `pin-confirm`에서 다른 PIN(`9999`) 입력 → "PIN이 일치하지 않습니다" 에러 표시 후 `pin-create`로 복귀
- [ ] 복귀한 `pin-create` 화면 입력창이 비어 있는지 확인 (이전 잘못된 값이 남지 않아야 함)
- [ ] `pin-create` 재진입 → 새 PIN(`5678`) 입력 → `pin-confirm` 진입 시에도 입력창이 비어 있어야 함
- [ ] `pin-confirm` 화면에서 [← BACK] 클릭 → 닉네임 화면 복귀
- [ ] 닉네임 화면에서 같은 닉네임 재입력 → `pin-create` 진입 시 입력창이 비어 있는지 확인
- [ ] 1자리·2자리·3자리만 입력한 상태에서 [확인] 버튼이 비활성화(`disabled`)되는지 확인
- [ ] 숫자가 아닌 키(영문, 특수문자) 입력 시 무시되는지 확인
- [ ] 5자리 이상 입력 시도 시 4자리에서 잘리는지 확인 (`maxLength=4`)

### 🔁 3. 화면 전환 회귀

- [ ] `pin-create` → `pin-confirm` → BACK → 닉네임 → 다른 닉네임 → `pin-create` (또는 `pin-login`) 시나리오에서 매 화면 진입마다 입력창이 초기화되는지 확인
- [ ] 기존 회원 로그인(`pin-login`) 흐름은 영향 없는지 확인 (PIN 입력 → 검증 성공/실패 정상 동작)
- [ ] `pin-login`에서 잘못된 PIN 입력 → "PIN이 틀렸습니다" 표시 후 입력창이 비워지는 기존 동작이 유지되는지 확인

### 🎨 4. UI / UX 확인

- [ ] PIN 입력창 placeholder(`● ● ● ●`)가 정상 표시
- [ ] 입력 중 글자가 마스킹(`type="password"`)되는지 확인
- [ ] 가이드 문구(`⚠️ PIN을 다시 한번 입력하세요…`)가 `pin-confirm` 단계에서만 노출되는지 확인
- [ ] 데스크탑 Chrome 기준 레이아웃 깨짐 없음
- [ ] 로딩 스피너(`loading-spinner`)가 확인 처리 중 정상 표시되는지 확인

### 🌐 5. 환경 매트릭스

- [ ] Windows + Chrome 최신
- [ ] Windows + Edge 최신
- [ ] macOS + Safari (가능 시)
- [ ] 모바일 경고 모달이 뜨는 환경(폭 < 768px)에서도 PIN 단계 전환 동작 확인
