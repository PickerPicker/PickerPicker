## 🧪 테스트 케이스: 모바일 접근 시 PC 권장 모달 표시 및 타이틀 반응형 폰트 적용
**이슈**: #67  **담당자**: @Cassiiopeia

---

### ✅ 1. 모달 표시 조건 확인

- [ ] 모바일 화면(viewport width < 768px)으로 접근 시 PC 권장 모달이 자동 표시된다
- [ ] PC 화면(viewport width ≥ 768px)으로 접근 시 모달이 표시되지 않는다
- [ ] 태블릿(예: 768px 정확히)에서 모달이 표시되지 않는다

### ✅ 2. 모달 UI 확인

- [ ] 모달에 SVG 모니터 아이콘이 표시된다
- [ ] 제목 "PC 플레이 권장"이 파란색(`#00b4ff`)으로 표시된다
- [ ] "키보드 조작 기반" 텍스트가 파란색 강조로 표시된다
- [ ] "PC 환경" 텍스트가 파란색 강조로 표시된다
- [ ] "키보드 조작 기반" 뒤 줄바꿈 후 "리듬 게임입니다."가 표시된다
- [ ] "확인" 버튼이 정상 렌더링된다
- [ ] 모달 배경이 블러 오버레이로 게임 화면을 가린다

### ✅ 3. 모달 dismiss 동작 확인

- [ ] "확인" 버튼 클릭 시 모달이 닫힌다
- [ ] 모달 닫은 후 게임 화면(StartScreen)이 정상 표시된다
- [ ] 모달 닫아도 게임 플레이에 제한이 없다

### ⚠️ 4. sessionStorage 중복 표시 방지

- [ ] 모달 닫은 후 같은 탭에서 페이지 내 이동/복귀 시 모달이 재표시되지 않는다
- [ ] 같은 탭에서 새로고침(F5) 시 모달이 **재표시된다** (sessionStorage는 새로고침 시 유지되므로 재표시 안 됨 — 실제 동작 확인)
- [ ] 탭을 닫고 새 탭으로 접근 시 모달이 다시 표시된다

### ✅ 5. h1 타이틀 반응형 폰트 확인

- [ ] 모바일(412px)에서 "PickerPicker" 타이틀이 `text-3xl` 크기로 표시되어 레이아웃을 벗어나지 않는다
- [ ] PC(768px 이상)에서 "PickerPicker" 타이틀이 `text-5xl` 크기로 표시된다
- [ ] 타이틀 텍스트 그림자(glow)가 양쪽 화면에서 정상 표시된다

### ⚠️ 6. 엣지 케이스

- [ ] 모달이 열린 상태에서 다른 모달(닉네임/PIN 입력 등)을 여는 동작이 발생하지 않는다 (모바일에서는 로그인 불필요 케이스)
- [ ] 모달 오버레이 영역(모달 카드 바깥) 클릭 시 닫히지 않는다 (의도된 동작 — "확인" 버튼만으로 dismiss)
- [ ] 매우 좁은 화면(320px)에서 모달 카드가 잘리지 않고 `max-w-sm` 내에서 정상 렌더링된다
