## 🧪 테스트 케이스: 로그인 플로우 인플레이스 전환
**이슈**: #66  **담당자**: @Cassiiopeia

---

### ✅ 1. 기본 기능 동작 확인

- [ ] 비로그인 상태에서 [시작] 클릭 시 모달 없이 카드 내부에 닉네임 입력 뷰가 표시됨
- [ ] 닉네임 입력 뷰에서 [← BACK] 클릭 시 홈 화면으로 복귀됨
- [ ] 기존 닉네임 입력 후 확인 시 `pin-login` 뷰(PIN 입력, 가이드라인 없음)로 전환됨
- [ ] 신규 닉네임 입력 후 확인 시 `pin-create` 뷰(노란 경고 박스 + "다음 로그인 시 필요" 문구)로 전환됨
- [ ] `pin-create` 뷰에서 [← BACK] 클릭 시 닉네임 입력 뷰로 복귀됨
- [ ] `pin-create` 4자리 PIN 입력 후 확인 시 `pin-confirm` 뷰("잊어버리면 재설정 불가" 문구)로 전환됨
- [ ] `pin-confirm` 뷰에서 [← BACK] 클릭 시 닉네임 입력 뷰로 복귀됨
- [ ] `pin-confirm`에서 PIN 일치 시 게임(또는 튜토리얼) 화면으로 진입됨
- [ ] `pin-login`에서 올바른 PIN 입력 시 게임 화면으로 진입됨
- [ ] 이미 로그인 상태에서 [플레이하기] 클릭 시 닉네임 뷰 없이 바로 게임 진입됨

### ⚠️ 2. 엣지 케이스 테스트

- [ ] 닉네임 입력 없이(빈 값) 확인 버튼 클릭 시 비활성 상태 유지(동작 없음)
- [ ] 닉네임 20자 초과 입력 불가(maxLength=20 적용 확인)
- [ ] `pin-login`에서 틀린 PIN 입력 시 "PIN이 틀렸습니다" 에러 메시지 표시 + PIN 입력창 초기화
- [ ] `pin-confirm`에서 PIN 불일치 시 `pin-create` 단계로 자동 복귀(에러 표시 없음)
- [ ] PIN 입력창에 숫자 외 문자 입력 불가(영문/특수문자 차단)
- [ ] PIN 3자리 이하 입력 상태에서 확인 버튼 비활성 확인
- [ ] 닉네임 API 호출 중 로딩 스피너 표시 + 확인/BACK 버튼 비활성화
- [ ] 공백만 입력한 닉네임은 trim 처리 후 빈 값으로 간주(버튼 비활성)

### 🎨 3. UI/UX 확인

- [ ] 닉네임/PIN 뷰가 홈 카드 내부(인플레이스)에서 표시되며 별도 overlay 모달 없음
- [ ] `pin-create`, `pin-confirm` 가이드라인 박스 노란 배경/테두리 스타일 정상 표시
- [ ] `pin-login` 뷰에는 가이드라인 박스 미표시 확인
- [ ] 각 뷰 진입 시 입력창 자동 포커스(autoFocus) 동작 확인
- [ ] Enter 키로 확인 동작(닉네임 뷰, PIN 뷰 모두)
- [ ] 설정/크레딧 화면과 동일한 ← BACK 버튼 스타일 적용 확인
- [ ] 480px 카드 내에서 레이아웃 깨짐 없음(모바일 90vw 포함)
