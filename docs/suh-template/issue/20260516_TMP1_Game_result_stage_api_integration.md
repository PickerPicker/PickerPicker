# ⚙️[기능추가][Game] 점수 판정 결과 저장 및 스테이지 API 연동

라벨: 작업전
담당자: Cassiiopeia

📝 현재 문제점
---

- 본 게임 플레이 중 판정 결과가 점수, 게이지, 콤보, 정확도, 최고 기록에 일관되게 반영되어야 합니다.
- 스테이지 클리어와 게임오버 시점에 결과 저장이 중복 호출되지 않도록 제어가 필요합니다.
- 프론트엔드 게임 화면과 백엔드 스테이지 데이터 API, 게임 결과 저장 API 사이의 실제 연동 흐름을 정리해야 합니다.
- 무효 음절 노트, 입력 타이밍, 결과 화면, 재시작 흐름이 하나의 게임 루프로 자연스럽게 이어져야 합니다.

🛠️ 해결 방안 / 제안 기능
---

- `PlayStage`에서 PERFECT/GOOD/MISS 판정 기준을 적용하고 점수, 게이지, 콤보, 최대 콤보, 판정 카운트를 갱신합니다.
- 무효 음절은 판정 없이 자동으로 진행하고, 게이지가 0 이하가 되면 게임오버 처리를 한 번만 실행합니다.
- `GameScreen`에서 스테이지 진행, 결과 화면, 정확도 계산, 로컬 최고 기록 갱신, 서버 결과 저장을 연결합니다.
- 전체 클리어와 게임오버 모두 최종 스탯을 기준으로 결과를 저장하고, 재시작 시 상태를 초기화합니다.
- 백엔드에 스테이지 데이터 조회 API를 추가해 1~15 스테이지 데이터셋을 제공할 수 있게 합니다.
- 관련 설계 문서에 게임 진행 플로우, 본 게임 화면, 판정/게이지/점수 시스템 변경 내용을 반영합니다.

⚙️ 작업 내용
---

- 프론트엔드 본 게임 판정 및 상태 갱신 로직 구현
- 결과 화면과 최고 기록 저장 로직 구현
- 게임 결과 저장 API 호출 연결
- 백엔드 스테이지 데이터 조회 라우터 등록
- 게임 진행 플로우 및 점수 시스템 설계 문서 정리
