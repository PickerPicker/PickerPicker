# 🚀[기능개선][로그인] 로그인 플로우 인플레이스 전환

**라벨**: 작업전  
**담당자**: Cassiiopeia

---

📝 현재 문제점
---

- 시작 버튼 클릭 시 닉네임 입력이 overlay `<dialog>` 모달로 표시됨
- PIN 입력도 별도 모달로 처리되어 설정/크레딧 화면과 UX 패턴이 불일치함
- 모달 방식은 뒤로가기(단계별 역행) 구현이 어렵고, 화면 전환 일관성이 없음
- 신규 유저가 PIN을 설정할 때 "나중에 다시 올 때 필요하다"는 안내가 없어 PIN을 잊을 위험이 있음

🛠️ 해결 방안 / 제안 기능
---

- 닉네임 입력 및 PIN 입력을 `StartScreen` 내부 인플레이스 뷰로 전환 (설정/크레딧 화면과 동일 패턴)
- `screen` state에 `nickname`, `pin-login`, `pin-create`, `pin-confirm` 추가
- 뒤로가기 단계별 지원: PIN 단계 → 닉네임 단계, 닉네임 단계 → 홈
- 신규 PIN 설정(`pin-create`, `pin-confirm`) 시 가이드라인 문구 표시:
  - `pin-create`: "⚠️ 이 PIN은 다음 로그인 시 필요합니다. 꼭 기억해 두세요!"
  - `pin-confirm`: "⚠️ PIN을 다시 한번 입력하세요. 잊어버리면 재설정이 불가합니다."
