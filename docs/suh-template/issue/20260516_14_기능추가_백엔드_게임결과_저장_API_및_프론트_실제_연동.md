📝 현재 문제점
---

- 백엔드 Player 모델이 score 하나만 저장하여 게임 결과 화면에 필요한 역대 최고 기록 표시 불가
- 프론트엔드 playerService.ts가 mock 데이터로 작동하여 실제 서버와 연동 없음
- 게임 종료 후 결과(점수, 스테이지, 콤보)를 서버에 저장하는 API 부재

🛠️ 해결 방안 / 제안 기능
---

- Player 모델에 best_score, best_stage, best_combo, play_count 필드 추가
- POST /players/result — 게임 종료 시 결과 저장 (각 항목 최고값만 갱신)
- GET /players/{nickname} — 역대 최고 기록 조회
- GET /ranking — best_score 기준 상위 랭킹 조회
- 프론트 playerService.ts mock 제거 후 실제 API 연결
- 닉네임 확인 시 신규 플레이어 자동 DB 등록

⚙️ 작업 내용
---

- backend/src/models/player.py — Player 모델 필드 확장
- backend/src/services/player_service.py — save_game_result 로직 구현
- backend/src/apis/player_router.py — /players/result 엔드포인트 추가
- backend/src/apis/ranking_router.py — best_score 기준 랭킹 조회
- src/services/playerService.ts — mock 제거, 실제 API 연결
- src/App.tsx — 신규 플레이어 createPlayer 자동 호출 연동
