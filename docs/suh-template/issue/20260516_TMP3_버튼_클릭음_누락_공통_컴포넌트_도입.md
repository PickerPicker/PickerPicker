# ❗[버그][UI] 홈/모달/랭킹 화면 일부 버튼 클릭음 누락 — 공통 컴포넌트 도입

🗒️ 설명
---

홈 화면, 닉네임/PIN/플레이어 타입/설정 모달, 랭킹 화면 등 게임 외 UI의 다수 버튼에서 클릭 시 `sfx_button.mp3` 효과음이 재생되지 않는다.

현재 `playButtonSfx` 함수는 `useAudio` 훅에 구현되어 있으나, `App.tsx`의 일부 핸들러(`handleStart`, `handleLogout`, ranking 진입, GameScreen `onButtonSfx` prop)에서만 명시적으로 호출되고 있고, 화면 내부 자식 버튼들은 호출이 누락된 상태다.

- 누락 위치 (확인됨):
  - `StartScreen`: 설정/크레딧 진입, ±오프셋 조정, 뒤로가기
  - `NicknameModal`, `PinModal`, `PlayerTypeModal`, `SettingsModal` 내부 버튼 전반
  - `RankingScreen` 뒤로가기 버튼
  - `GameScreen` 일부 버튼 (게임 키보드 제외)

🔄 재현 방법
---

1. 홈 화면 진입
2. 우측 상단 설정 버튼 클릭 → **클릭음 없음**
3. 닉네임 모달에서 X 닫기 클릭 → **클릭음 없음**
4. 랭킹 화면 진입 후 뒤로가기 클릭 → 클릭음 정상
5. 설정 모달 안의 ± 오프셋 버튼 클릭 → **클릭음 없음**

✅ 예상 동작
---

- 게임 외 UI의 **모든 button 요소** 클릭 시 `sfx_button.mp3` 재생
- 게임 진행 중 키보드(노트 키) 버튼은 노트 사운드와 충돌하므로 **명시적으로 무음 처리**
- 클릭음 호출이 누락될 수 없도록 공통 컴포넌트로 강제

🛠️ 해결 방안
---

1. **`SoundButton`** 공통 컴포넌트 도입
   - `<button>` 래퍼. `onClick` 호출 직전 `playButtonSfx` 자동 실행
   - `sfx` prop으로 재생할 SFX 종류 지정 가능 (기본 `'button'`, 추후 확장)
   - 모든 `className`/`disabled`/기타 native button props 통과

2. **`SilentButton`** 공통 컴포넌트 도입
   - 게임 내부 키보드 버튼 전용
   - 클릭음 명시적 무음
   - 의도 명시 → grep으로 누락 검사 가능

3. **`AudioContext` Provider** 도입
   - 기존 `useAudio()` 결과를 Context로 제공
   - `useAudioContext()` 훅으로 어디서든 접근 → prop drilling 제거
   - `App.tsx`의 `onButtonSfx` prop 체인 제거

4. 적용 범위
   - `StartScreen`, `NicknameModal`, `PinModal`, `PlayerTypeModal`, `SettingsModal`, `RankingScreen`, `GameScreen`: `<button>` → `<SoundButton>`
   - `PlayStage`, `KeyboardDisplay` (게임 내 키보드): `<button>` → `<SilentButton>`
