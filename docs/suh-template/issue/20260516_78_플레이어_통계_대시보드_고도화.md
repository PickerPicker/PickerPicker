📝 현재 문제점
---

- 현재 `players` 테이블에는 `best_score / best_stage / best_combo / play_count` 등 **최고값 누적치**만 저장되어 개별 플레이 기록(시계열, 평균, 분포)을 확인할 수 없다.
- 사용자가 본인 실력의 **추세**(7일/30일 평균, 일별 추이)나 **전체 대비 위치**(백분위, 상위 N%)를 알 방법이 없다.
- 스테이지별 도달 통계(스테이지별 최고점/도달률) 같은 다음 목표 설정에 필요한 데이터가 부재하다.
- 결과 화면에서 "역대 최고 기록"만 보여주는 단편적 피드백 → 재플레이 동기 부여 약화.

🛠️ 해결 방안 / 제안 기능
---

- **백엔드 데이터 모델 확장**
  - `game_sessions` 테이블 신설: 플레이 1회당 1행. `nickname`, `score`, `stage`, `combo`, `stage_scores` (JSONB: `{1: 1200, 2: 950, ...}`), `played_at`. **30일 보존**.
  - `player_stats_daily` 테이블 신설: `(nickname, date)` 복합 PK. `play_count`, `sum_score`, `max_score`, `max_stage`, `max_combo`. 영구 보존.
  - `player_sessions` 테이블 신설: 인증 세션 토큰 저장 (`token`, `nickname`, `expires_at`). 24시간 TTL.

- **백엔드 통계 로직**
  - `POST /players/result`: 세션 INSERT + 일별 집계 UPSERT를 단일 트랜잭션으로 처리. `stage_scores` 필드 추가.
  - `POST /auth/login`: PIN 검증 후 세션 토큰 발급.
  - `GET /players/{nickname}/stats` (인증 필요): 본인 기본 집계 + 시계열 + 백분위 + 스테이지별 최고점.
  - `GET /stats/global` (공개): 전체 평균/중앙값/총 플레이 수. 5분 인메모리 캐시.
  - `GET /players/{nickname}/sessions?days=30` (인증 필요): 일별 시계열.
  - 30일 초과 세션 자동 정리 (FastAPI startup background task 또는 cron).

- **프론트엔드 통계 화면 (`StatsScreen.tsx` 신설)**
  - 진입점 2개: 결과 화면 "내 통계 보기" 버튼 + 메인 화면 "내 통계" 버튼.
  - 섹션 구성:
    1. 기본 집계 카드 (평균 score, 평균 stage, 평균 combo, 총 플레이 수, 최고/최저/중앙값)
    2. 시계열 차트 (7일/30일 이동 평균, 일별 플레이 횟수)
    3. 전체 대비 순위 (백분위, 상위 N%)
    4. 스테이지별 성과 표 (1~15 스테이지, 최고점, 도달률, 마지막 플레이 일시)
    5. 관습/분포 (시간대별 플레이 히트맵, 세션 간격 분포)
  - 로그인 안 된 상태 진입 → PIN 입력 → 세션 토큰 발급 → 통계 로드.
