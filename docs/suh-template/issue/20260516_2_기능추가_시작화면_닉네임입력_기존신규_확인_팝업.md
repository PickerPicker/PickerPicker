📝 현재 문제점
---

- 게임 진입 시 사용자를 식별할 수 있는 시작 화면과 닉네임 입력 흐름이 없음
- 기존 플레이어와 신규 플레이어를 구분하는 UX 로직 부재

🛠️ 해결 방안 / 제안 기능
---

- **시작 화면**: 게임 제목 텍스트, `랭킹` 버튼(랭킹 화면으로 이동), `시작` 버튼(닉네임 입력 팝업 오픈)
- **닉네임 입력 팝업**: 텍스트 입력창 + `확인` 버튼
  - 확인 클릭 시 서버(임시: 로컬 mock)에 닉네임 존재 여부 확인
  - 기존 닉네임 → 기존/신규 확인 팝업 표시
  - 신규 닉네임 → 바로 게임 화면으로 이동
- **기존/신규 확인 팝업**: "이미 존재하는 닉네임입니다. 신규 플레이어로 플레이하시겠습니까?" 메시지
  - `기존 플레이어로 플레이` 버튼 → 기존 데이터로 게임 화면 접속
  - `신규 플레이어로 플레이` 버튼 → 새 플레이어로 게임 화면 접속

⚙️ 작업 내용
---

- 시작 화면 컴포넌트 구현 (게임 제목, 랭킹 버튼, 시작 버튼)
- 닉네임 입력 팝업 컴포넌트 구현
- 기존/신규 확인 팝업 컴포넌트 구현
- 닉네임 존재 여부 확인 로직 구현 (서버 없이 로컬호스트 mock 처리)
- 화면 전환 라우팅 연결
