# 스테이지 데이터 구조

## 개요

개별 스테이지(실제 플레이 라운드)와 난이도 단계(여러 스테이지를 묶는 구간)를 구분한다.

```
스테이지 = 실제 플레이하는 개별 라운드
난이도 단계 = 3개 스테이지를 묶는 난이도 구간 (Difficulty Tier)
```

---

## 전체 구성

총 15스테이지, 5개의 난이도 단계, 3스테이지마다 난이도 상승.

| 난이도 단계 | 스테이지 범위 | 명칭 |
| --- | --- | --- |
| 1단계 | Stage 1~3 | 입문 |
| 2단계 | Stage 4~6 | 확장 |
| 3단계 | Stage 7~9 | 8키 진입 |
| 4단계 | Stage 10~12 | 복수 단어 |
| 5단계 | Stage 13~15 | 고난도 |

---

## 난이도 단계별 특징

### 1단계: 입문 (Stage 1~3)

- 단어 1개, 4키 사용
- 짧은 리듬 큐, 느린 BPM
- 방해 음절 최소

### 2단계: 확장 (Stage 4~6)

- 단어 1개, 6키 사용
- 리듬 큐 증가, BPM 증가
- 헷갈리는 음절 추가

### 3단계: 8키 진입 (Stage 7~9)

- 단어 1개, 8키 전체 사용
- 리듬 큐 길어짐, BPM 추가 증가
- 비슷한 발음의 음절 증가

### 4단계: 복수 단어 (Stage 10~12)

- 단어 2개, 8키 사용
- 두 단어의 음절이 섞여서 나옴

### 5단계: 고난도 (Stage 13~15)

- 단어 2~3개, 8키 사용
- 긴 리듬 큐, 빠른 BPM
- 비슷한 발음 다수, MISS 시 치명적

## 계산 공식

```
difficultyLevel = floor((stage - 1) / 3) + 1
bpm = 90 + (difficultyLevel - 1) * 15
inputLength = 16 + (difficultyLevel - 1) * 8
```

## 키보드 키 활성화

키는 8개 고정(`A S D F J K L ;`)이며, 초반 스테이지에서는 일부만 활성화한다.

| 구간 | 활성 키 |
| --- | --- |
| 초반 (1단계) | A S D F (4키) |
| 중반 (2단계) | A S D F J K (6키) |
| 후반 (3~5단계) | A S D F J K L ; (8키) |

화면에서는 8칸 레이아웃을 유지하고, 미활성 키를 흐리게 표시한다:

```
[A] [S] [D] [F]    [J] [K] [L] [;]
 커  피  쿠  푸     -   -   -   -
```

---

## 단어 표시 규칙 (2개 이상)

단어가 2개 이상인 경우 **줄바꿈으로 세로 나열**한다.

```
이번 단어

커피
라면
```

단어가 3개인 경우:

```
이번 단어

커피
라면
바나나
```

---

## 복수 단어 스테이지의 입력 음절 구성

두 단어의 음절 전체가 입력 음절에 포함되어야 한다.

```
이번 단어: 커피 / 라면
정답 음절: 커, 피, 라, 면
방해 음절: 쿠, 푸, 러, 먼

입력 음절 (8개): [커] [피] [라] [면] [쿠] [푸] [러] [먼]
```

리듬 큐는 두 단어의 음절이 섞여서 구성된다:

```
[커] [커] [피] [피] [라] [라] [면] [면]
또는 (더 어려운 버전)
[커] [라] [커] [피] [면] [라] [피] [면]
```

## 스테이지 데이터 구조 (JSON 스키마)

```json
{
  "stage": 1,
  "difficultyLevel": 1,
  "bpm": 90,
  "word": "커피",
  "validSyllables": ["커", "피"],
  "invalidSyllables": ["코", "포", "키", "퍼", "카", "파"],
  "inputLength": 16,
  "inputSyllables": ["커", "피", "커", "코", "피", "커", "피", "포", "커", "피", "커", "키", "피", "커", "피", "퍼"],
  "keyMapping": [
    { "key": "a", "syllable": "커", "type": "valid" },
    { "key": "s", "syllable": "피", "type": "valid" },
    { "key": "d", "syllable": "코", "type": "invalid" },
    { "key": "f", "syllable": "포", "type": "invalid" },
    { "key": "j", "syllable": "키", "type": "invalid" },
    { "key": "k", "syllable": "퍼", "type": "invalid" },
    { "key": "l", "syllable": "카", "type": "invalid" },
    { "key": ";", "syllable": "파", "type": "invalid" }
  ]
}
```

---

## 데이터셋 생성 규칙

1. `difficultyLevel = floor((stage - 1) / 3) + 1`
2. `bpm = 90 + (difficultyLevel - 1) * 15`
3. `inputLength = 16 + (difficultyLevel - 1) * 8`
4. `keyMapping`은 `keyLayout` 순서와 1:1 대응, 길이 항상 8
5. `validSyllables` 개수: 난이도 1~2는 2개, 3~4는 3개, 5 이상은 4개
6. `invalidSyllables` 개수 = 8 - validSyllables.length
7. `invalidSyllables`는 `validSyllables`와 자음을 공유하되 동일하면 안 됨
8. `inputSyllables` 내 유효 음절 비율 70% 이상
9. 같은 단어는 재사용하지 않음

---

## 프론트 / 백엔드 사용법

```
프론트: stage.bpm, stage.inputSyllables, stage.keyMapping 사용
입력 처리: keyMapping에서 event.code로 syllable 조회 → inputSyllables[index]와 비교

백엔드: stages 배열 저장 또는 stage 단위 API 응답
```

키 입력 처리 시 `event.code` 사용 (한글 입력 상태에서도 정상 동작):

```js
document.addEventListener("keydown", (event) => {
  const syllable = stage.keyMapping.find(k => k.key === event.code.toLowerCase())?.syllable;
  if (!syllable) return;
  handleInput(syllable);
});
```
