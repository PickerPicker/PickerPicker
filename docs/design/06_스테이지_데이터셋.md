## 키매핑 배열 (전 스테이지 공통 구조)

키매핑 배열은 `keyLayout` 순서와 1:1 대응한다:

```
[A, S, D, F, J, K, L, ;]
```

유효 음절이 앞 순서에, 무효 음절이 뒤 순서에 배치된다.

### 예시 (Stage 1 — 커피)

```
키매핑: 커, 피, 코, 포, 키, 퍼, 카, 파
A=커  S=피  D=코  F=포  J=키  K=퍼  L=카  ;=파
```

### 예시 (Stage 7 — 하늘빛)

```
키매핑: 하, 늘, 빛, 호, 너, 보, 해, 노
A=하  S=늘  D=빛  F=호  J=너  K=보  L=해  ;=노
```

---

## 무효 음절 생성 규칙

- 유효 음절과 **자음이 겹쳐야** 하지만 **동일하면 안 됨**
- 8 - validSyllables.length 개만큼 생성

예시 (유효 음절 `커, 피`):

```
유효: 커(ㅋ), 피(ㅍ)
무효: 코(ㅋ), 카(ㅋ), 키(ㅋ), 퍼(ㅍ), 포(ㅍ), 파(ㅍ)  ← 자음 겹침
```

## 추가 스테이지 생성 프롬프트

16~30스테이지 등 추가 데이터가 필요할 때 사용할 GPT 프롬프트:

```
아래 규칙에 맞춰 리듬게임 스테이지 JSON 데이터를 생성해라.
반드시 유효한 JSON만 출력하고, 설명/주석/마크다운은 출력하지 마라.

생성 범위: stage 16 ~ 30

규칙:
1. difficultyLevel = floor((stage - 1) / 3) + 1
2. bpm = 90 + (difficultyLevel - 1) * 15
3. inputLength = 16 + (difficultyLevel - 1) * 8
4. keyLayout: ["a", "s", "d", "f", "j", "k", "l", ";"] 고정
5. keyMapping 길이 항상 8
6. validSyllables 개수: 난이도 1~2=2개, 3~4=3개, 5이상=4개
7. invalidSyllables = 8 - validSyllables.length
8. invalidSyllables는 validSyllables와 자음 공유, 동일 불가
9. inputSyllables.length === inputLength
10. inputSyllables 내 유효 음절 비율 70% 이상
11. 단어 중복 없음

이전 사용 단어:
["커피", "바다", "나무", "사과", "토끼", "구름", "하늘빛", "노래방", "바람결",
 "초콜릿", "달고나", "버섯국", "컴퓨터실", "리듬게임", "해커톤장"]
```
