# 리듬 미리보기 화면

## 개요

각 스테이지 플레이 직전에 표시되는 화면으로, 사용자가 이번 스테이지의 단어·리듬·키보드 매핑을 미리 확인하고 학습하는 단계다.
사용자는 키를 직접 누르지 않으며, 리듬 큐와 키보드 매핑이 박자에 맞춰 자동으로 반짝인다.

---

## 화면 레이아웃

```
┌──────────────────────────────────────────┐
│  STAGE 1          LIFE ████████░░  ░░░░  │
├──────────────────────────────────────────┤
│                 이번 단어                 │
│                   커피                    │
│                                          │
│                 입력 음절                 │
│   [커] [쿠] [카] [피] [푸] [퍼] [파] [포] │
│                                          │
│                 리듬 큐                   │
│   [커] [커] [커] [커] [피] [피]           │
│     ↑  (현재 박자 강조)                   │
│                                          │
│               키보드 매핑                 │
│   [A] [S] [D] [F]    [J] [K] [L] [;]    │
│    커  쿠  카  피     푸  퍼  파  포      │
└──────────────────────────────────────────┘
```

---

## 영역별 명세

### 1. 상단 상태 바

- 현재 스테이지 번호 (`STAGE N`)
- 생존 게이지 바 (항상 표시)

### 2. 이번 단어

- 이번 라운드에서 사용할 단어를 표시
- 단어가 2개 이상일 경우 **반드시 줄바꿈**하여 세로로 나열

```
이번 단어

커피
라면
```

> 한 줄에 `커피 / 라면` 형식으로 나열하지 않는다.

### 3. 입력 음절

- 유효 음절(정답)과 무효 음절(방해)을 구분하지 않고 함께 표시
- 사용자에게는 전부 "입력 가능한 음절"로 보여줌

```
입력 음절
[커] [쿠] [카] [피] [푸] [퍼] [파] [포]
```

초반 스테이지에서 일부 키만 활성화할 경우 비활성 키는 흐리게 처리:

```
[A] [S] [D] [F]    [J] [K] [L] [;]
 커  피  쿠  푸     -   -   -   -
```

### 4. 리듬 큐

- 사용자가 이번 스테이지에서 처리해야 할 음절 순서
- 미리보기 단계에서는 현재 박자에 해당하는 음절이 강조(반짝임)됨

```
[커] [커] [커] [커] [피] [피]
  ↑
```

다음 박자:

```
[커] [커] [커] [커] [피] [피]
       ↑
```

### 5. 키보드 매핑

- 8키 고정 (`A S D F` / `J K L ;`)
- 각 키에 매핑된 음절 표시
- 미리보기 중 리듬 큐와 동기화하여 해당 키도 함께 반짝임

```
리듬 큐 현재 음절: 커  →  [A] 버튼 반짝
리듬 큐 현재 음절: 피  →  [F] 버튼 반짝
```

---

## 미리보기 애니메이션 동작

1. 리듬 큐의 첫 번째 음절부터 BPM에 맞춰 순서대로 강조
2. 강조와 동시에 해당 음절이 매핑된 키보드 키도 함께 반짝임
3. 전체 리듬 큐를 1회 재생 후 플레이 화면으로 전환

## 용어 정리

| 화면 표시 용어 | 내부 개발 용어 |
| --- | --- |
| 이번 단어 | `word` |
| 입력 음절 | `inputSyllables` (= `validSyllables` + `invalidSyllables`) |
| 리듬 큐 | `rhythmQueue` |
| 키보드 매핑 | `keyMapping` |
