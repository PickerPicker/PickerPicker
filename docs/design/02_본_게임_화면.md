# 본 게임 화면

## 화면 구성

게임 화면은 3단으로 구성된다.

```
┌────────────────────────────────────────────┐
│ Stage 1   LIFE ████████░░  Score 000000    │
│           이번 단어: 커피       3 COMBO     │
├────────────────────────────────────────────┤
│                                            │
│              처리해야 할 음절 배열          │
│      [커][커][커][커][피][피][커][피]       │
│                                            │
│          PERFECT / GOOD / MISS             │
│                                            │
├────────────────────────────────────────────┤
│                키보드 매핑                  │
│   [A] [S] [D] [F]    [J] [K] [L] [;]      │
│    커  쿠  카  피     푸  퍼  파  포        │
└────────────────────────────────────────────┘
```

---

## 상단 영역

| 요소 | 설명 |
| --- | --- |
| 현재 스테이지 | `Stage N` 표시 |
| 생존 게이지 | 시각적 바(bar) 형태, 숫자 미표시 |
| 현재 점수 | `Score: XXXXXX` (6자리 고정) |
| PERFECT 콤보 | 1 이상일 때만 `N COMBO` 표시 |
| 이번 단어 | 음절별 박스로 표시 |

게이지 색상:
- 50% 초과: 기본 (primary)
- 25~50%: 경고 (warning/주황)
- 25% 이하: 위험 (error/빨강)

---

## 중앙 영역 (플레이 영역)

노트가 오른쪽에서 왼쪽으로 이동하며 판정선에 도달했을 때 키를 입력한다.

```
판정선 │        ← [커]     ← [커]     ← [피]
```

처리해야 할 음절 배열 전체를 위에 표시하고, 현재 처리 중인 음절을 강조 표시한다.

판정 결과(`PERFECT` / `GOOD` / `MISS`)는 중앙에 크게 표시된 후 사라진다.

---

## 하단 키보드 매핑 영역

8키 고정 레이아웃:

```
[A] [S] [D] [F]    [J] [K] [L] [;]
 커  쿠  카  피     푸  퍼  파  포
```

키 입력 시 해당 키가 시각적으로 눌린 효과를 보여준다.

---

## 입력 규칙

```
A → 1번째 음절
S → 2번째 음절
D → 3번째 음절
F → 4번째 음절
J → 5번째 음절
K → 6번째 음절
L → 7번째 음절
; → 8번째 음절
```

키 입력 시 확인하는 두 가지 조건:
1. 눌러야 하는 음절과 입력한 음절이 일치하는가
2. 박자 타이밍에 맞게 눌렀는가

둘 다 충족해야 PERFECT/GOOD 판정.

---

## 리듬 미리보기 화면

플레이 시작 전 2초 동안 표시된다.

```
STAGE 4

LIFE ███████░░░

이번 단어

커피라면

리듬 큐
[커] [라] [커] [피] [면] [라] [피] [면]

키보드
[A] [S] [D] [F]    [J] [K] [L] [;]
 커  피  라  면     쿠  푸  러  먼
```

미리보기 후 자동으로 플레이로 전환된다.

---

## 판정 시스템 요약

| 판정 | 조건 | 결과 |
| --- | --- | --- |
| PERFECT | 정확한 음절 + ±70ms 이내 | 점수+100, 게이지+1, 콤보+1 |
| GOOD | 정확한 음절 + ±140ms 이내 | 점수+50, 게이지 유지, 콤보 초기화 |
| MISS | 틀린 음절 또는 타이밍 초과 | 게이지-15, 콤보 초기화 |

5연속 PERFECT 달성 시 강화 보상: 점수+200, 게이지+2.
