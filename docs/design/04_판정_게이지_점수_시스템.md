# 판정 · 게이지 · 점수 시스템

## 시스템 개요

**지속형 생존 게이지 시스템**을 사용한다.

- 게이지는 스테이지마다 초기화되지 않고 다음 스테이지로 유지된다.
- PERFECT를 많이 칠수록 게이지를 보존하며 더 멀리 진행할 수 있다.
- 게이지는 생존 판단에, 점수는 결과 화면 평가에 사용한다.

> 게이지는 생존용, 점수는 결과용.

---

## 판정 종류

| 판정 | 조건 |
| --- | --- |
| PERFECT | 정확한 타이밍(±70ms)에 정확한 음절 입력 |
| GOOD | 약간 빠르거나 늦었지만(±140ms) 정확한 음절 입력 |
| MISS | 틀린 음절 입력 또는 타이밍 초과 |

---

## 판정별 보상표 (MVP)

| 판정 | 게이지 변화 | 점수 | PERFECT 콤보 |
| --- | --- | --- | --- |
| PERFECT (콤보 < 5) | +1 | +100 | +1 |
| PERFECT (콤보 ≥ 5) | +2 | +200 | +1 |
| GOOD | 0 | +50 | 초기화 |
| MISS | -15 | 0 | 초기화 |

---

## PERFECT 콤보

콤보는 **연속 PERFECT 횟수**만 카운트한다.

- GOOD도 콤보를 끊는다. (완벽한 리듬을 유지하지 못한 것으로 처리)
- MISS도 동일하게 콤보를 끊는다.

```
PERFECT → 콤보 증가
GOOD    → 콤보 0으로 초기화
MISS    → 콤보 0으로 초기화
```

5연속 PERFECT부터 강화 보상 적용:

| 연속 PERFECT | 게이지 | 점수 |
| --- | --- | --- |
| 1~4회 | +1 | +100 |
| 5회 이상 | +2 | +200 |

---

## 게이지 규칙

```
시작 게이지: 100
최대 게이지: 100
최소 게이지: 0
스테이지 간 게이지 유지 (초기화 없음)
```

게이지가 0이 되면 즉시 게임 오버.

---

## 게이지 흐름 예시

```
Stage 1 시작: 게이지 100
Stage 1 종료: 게이지 83
Stage 2 시작: 게이지 83  ← 유지
Stage 2 종료: 게이지 41
Stage 3 시작: 게이지 41  ← 유지
Stage 3 중 게이지 0 → 게임 오버
```

## 플레이 예시

리듬 큐: `커 커 커 커 피 피`

입력 결과가 아래와 같을 때:

| 순서 | 판정 | PERFECT 콤보 | 게이지 변화 | 점수 |
| --- | --- | --- | --- | --- |
| 1 | PERFECT | 1 | +1 | +100 |
| 2 | PERFECT | 2 | +1 | +100 |
| 3 | PERFECT | 3 | +1 | +100 |
| 4 | PERFECT | 4 | +1 | +100 |
| 5 | PERFECT | 5 | +2 | +200 |

결과: 게이지 +6, 점수 +650, 콤보 GOOD에서 끊김

## 스테이지 길이별 게이지 보상 (고도화)

리듬 큐가 짧은 스테이지는 PERFECT 기회가 적으므로 게이지 보상을 조정한다.

| 리듬 큐 길이 | 기본 PERFECT 게이지 | 5콤보 PERFECT 게이지 |
| --- | --- | --- |
| 4~6개 (짧음) | +2 | +3 |
| 7~12개 (일반) | +1 | +2 |
| 13개 이상 (김) | +1 | +1 |

> 1차 구현에서는 모든 스테이지 공통 판정표를 사용하고, 2차 밸런싱 단계에서 길이별 조정을 추가한다.

## 판정 처리 로직 (의사 코드)

```js
function applyJudgeResult(judge) {
  if (judge === "PERFECT") {
    perfectCombo += 1;
    if (perfectCombo >= 5) {
      gauge += 2;
      score += 200;
    } else {
      gauge += 1;
      score += 100;
    }
  }

  if (judge === "GOOD") {
    perfectCombo = 0;
    score += 50;
  }

  if (judge === "MISS") {
    perfectCombo = 0;
    gauge -= 15;
  }

  gauge = Math.min(gauge, 100);

  if (gauge <= 0) {
    gameOver();
  }
}
```

## 판정 타이밍 기준 (권장 시작값)

| 판정 | 허용 범위 |
| --- | --- |
| PERFECT | ±70ms (60fps 기준 ±4프레임) |
| GOOD | ±140ms |
| MISS | 그 외 |

> 추후 스테이지 난이도에 따라 허용 범위를 좁히는 방식으로 조정 가능.

---

## 정확도 계산

결과 화면에 표시하는 정확도:

```
정확도 (%) = (PERFECT + GOOD) / (PERFECT + GOOD + MISS) * 100
```

PERFECT 단독 비율:

```
PERFECT RATE (%) = PERFECT / (PERFECT + GOOD + MISS) * 100
```

## 최대 콤보 (maxCombo)

게임 중 달성한 PERFECT 연속 최고 횟수를 별도로 기록한다.
결과 화면 및 localStorage에 저장된다.

```
콤보 증가 시마다: maxCombo = max(maxCombo, perfectCombo)
```
