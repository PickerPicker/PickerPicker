## 🧪 테스트 케이스: 게임 화면 playCount DB 동기화
**이슈**: #40  **담당자**: @Cassiiopeia  **PR**: #60

배포 환경에서 검증. 백엔드 응답의 `play_count`가 게임오버/랭킹 화면에 정확히 반영되는지 확인.

---

### ✅ 1. 기본 기능 동작 확인

#### 게임오버 화면 (1회 플레이)
- [ ] 신규 닉네임 가입 → 1게임 플레이 완료 → 게임오버 화면 "플레이 횟수" = `1회` 표시
- [ ] 게임오버 화면 진입 직후 → 서버 응답 도착 전 `...` placeholder 표시 확인
- [ ] 서버 응답 도착 후 → `N회` 로 정상 갱신

#### 게임오버 화면 (누적 플레이)
- [ ] 같은 닉네임으로 3회 플레이 → 3번째 종료 화면에서 `3회` 표시
- [ ] 같은 닉네임으로 10회 플레이 → `10회` 표시
- [ ] "다시 하기" 버튼 → 재플레이 → 게임오버 화면에 `+1` 된 값 표시

#### 다른 브라우저/기기 동기화 (핵심 회귀 케이스)
- [ ] 브라우저 A에서 5회 플레이 (DB `play_count = 5`)
- [ ] 브라우저 B (또는 localStorage 비운 시크릿창)에서 동일 닉네임 로그인 → 1회 플레이 → 게임오버 화면 `6회` 표시 (이전 버그: `1회`로 표시되던 문제)
- [ ] localStorage 강제 삭제 후 재플레이 → DB 누적 값 표시 확인 (로컬 카운터 의존성 제거 검증)

#### 랭킹 화면
- [ ] 랭킹 화면 진입 → 테이블에 "플레이" 컬럼 존재 확인
- [ ] 각 랭킹 행에 `N회` 단위로 플레이 횟수 표시
- [ ] 컬럼 순서: 순위 / 닉네임 / 점수 / 스테이지 / 콤보 / 플레이
- [ ] 플레이 횟수가 0인 신규 플레이어도 누락 없이 표시

---

### ⚠️ 2. 엣지 케이스

#### 네트워크 / API 실패
- [ ] 백엔드 다운 상태에서 게임 종료 → 게임오버 화면 진입 → "플레이 횟수" `...` 계속 표시 (앱 크래시 없음)
- [ ] `/players/result` 응답 지연 (느린 네트워크) → 응답 도착 시점에 `...` → `N회` 갱신
- [ ] `/ranking` 응답 실패 → 랭킹 화면 빈 상태 메시지 표시 (기존 동작 유지)

#### 데이터 정합성
- [ ] DB `play_count = 0` 인 신규 플레이어 첫 게임 → 게임오버 화면 `1회` 표시 (서버가 증가 후 응답)
- [ ] 닉네임에 한글/특수문자 포함 → playCount 정상 조회/표시
- [ ] 매우 큰 누적 값 (예: 999회) → 화면 깨짐 없이 표시

#### 게임 시나리오
- [ ] STAGE 1 즉시 게임 오버 → playCount 증가 + 화면 표시 정상
- [ ] STAGE 15 ALL CLEAR → playCount 증가 + 화면 표시 정상
- [ ] "다시 하기" 연타 → playCount 중복 증가 없이 1회만 카운트 (resultSavedRef 동작 확인)

#### localStorage 관련 (회귀)
- [ ] localStorage 손상된 JSON → 앱 정상 시작, 게임오버 화면 정상 표시
- [ ] localStorage `playCount` 필드가 남아있어도 화면에 영향 없음 (필드 제거됨)
